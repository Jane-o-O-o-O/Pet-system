# 宠物信息管理系统使用说明

## 1. 项目简介

本项目是一个基于 **Spring Boot（后端） + Vue3 + Element Plus（前端） + MySQL（数据库）** 的前后端分离宠物信息管理系统。系统包含管理员 (Admin)、服务人员 (Staff) 和宠物主人 (Owner) 三种角色，并实现了严格的权限隔离，包括医疗记录与疫苗提醒、寄养订单管理、以及管理员统计报表等核心功能。

## 2. 技术栈

### 后端

*   **Spring Boot 3.x**: 快速开发框架。
*   **Spring Security + JWT**: 用户认证与授权。
*   **MyBatis-Plus**: 简化数据库操作。
*   **MySQL 8.x**: 关系型数据库。
*   **Spring Scheduler**: 定时任务，用于疫苗到期提醒。

### 前端

*   **Vue 3 + Vite**: 渐进式JavaScript框架及快速构建工具。
*   **Element Plus**: 基于 Vue 3 的桌面端组件库。
*   **Pinia**: Vue 3 的状态管理库。
*   **Axios**: 基于 Promise 的 HTTP 客户端。

## 3. 环境准备

在运行本项目之前，请确保您的开发环境已安装以下软件：

*   **Java Development Kit (JDK) 17 或更高版本**
*   **Maven 3.x 或更高版本**
*   **Node.js 18.x 或更高版本**
*   **pnpm 8.x 或更高版本** (推荐使用 pnpm 作为包管理器)
*   **MySQL 8.x 数据库**

## 4. 部署与运行

### 4.1 数据库配置

1.  **创建数据库**: 在 MySQL 中创建一个名为 `pet_mgmt` 的数据库。
    ```sql
    CREATE DATABASE IF NOT EXISTS pet_mgmt DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
    ```
2.  **导入初始数据**: 运行后端项目 `backend/src/main/resources/init.sql` 脚本，创建所有必要的表并插入初始管理员数据。
    *   默认管理员账号：`admin`
    *   默认管理员密码：`admin123`

### 4.2 后端项目 (Spring Boot)

1.  **进入后端目录**:
    ```bash
    cd /home/ubuntu/pet-management-system/backend
    ```
2.  **配置数据库连接**: 编辑 `src/main/resources/application.yml` 文件，根据您的 MySQL 配置修改 `spring.datasource.username` 和 `spring.datasource.password`。
    ```yaml
    spring:
      datasource:
        url: jdbc:mysql://localhost:3306/pet_mgmt?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=GMT%2B8&allowPublicKeyRetrieval=true
        username: root
        password: your_mysql_password # 修改为您的MySQL密码
    ```
3.  **编译并运行**: 
    ```bash
    mvn clean install
    java -jar target/pet-mgmt-0.0.1-SNAPSHOT.jar
    ```
    或者直接在 IDE (如 IntelliJ IDEA) 中运行 `PetMgmtApplication.java`。

    后端服务默认运行在 `http://localhost:8080`。

### 4.3 前端项目 (Vue3)

1.  **进入前端目录**:
    ```bash
    cd /home/ubuntu/pet-management-system/frontend
    ```
2.  **安装依赖**:
    ```bash
    pnpm install
    ```
3.  **运行开发服务器**:
    ```bash
    pnpm dev
    ```
    前端开发服务器默认运行在 `http://localhost:5173` (或其他端口，请查看控制台输出)。前端已配置代理，会将 `/api` 请求转发到后端 `http://localhost:8080`。

4.  **构建生产版本**:
    ```bash
    pnpm build
    ```
    构建后的文件将生成在 `dist` 目录下，可用于生产环境部署 (例如使用 Nginx 反向代理)。

## 5. 角色与功能

### 5.1 管理员 (Admin)

*   **登录账号**: `admin`，**密码**: `admin123`
*   **功能**: 
    *   用户管理：查看、禁用、启用用户，修改用户角色 (Admin/Staff/Owner)。
    *   宠物管理：查看所有宠物信息。
    *   统计报表：查看系统概览、寄养订单统计、疫苗到期统计。
    *   系统配置：管理疫苗提醒默认天数、定时任务 cron 表达式 (可选)。

### 5.2 服务人员 (Staff)

*   **登录账号**: 需由管理员创建。
*   **功能**: 
    *   宠物管理：查看所有宠物信息。
    *   医疗记录管理：为宠物新增、修改、删除医疗记录。
    *   疫苗记录管理：为宠物新增、修改、删除疫苗记录。
    *   寄养订单处理：确认订单、开始寄养、完成寄养、取消订单。

### 5.3 宠物主人 (Owner)

*   **登录账号**: 可自行注册，或由管理员/服务人员创建。
*   **功能**: 
    *   注册与登录。
    *   我的宠物：管理自己的宠物档案 (增删改查)。
    *   我的订单：查看自己宠物的寄养订单，申请寄养，取消未完成订单。
    *   查看自己宠物的医疗记录和疫苗记录。
    *   提醒中心：接收疫苗到期提醒等站内通知。

## 6. 注意事项

*   **密码加密**: 用户密码在数据库中存储的是 BCrypt 加密后的哈希值，不可逆。
*   **JWT 认证**: 系统使用 JWT 进行用户认证，Token 有效期为 7 天。
*   **权限控制**: 后端实现了基于角色的权限控制 (RBAC) 和数据级权限控制，确保不同角色只能访问其被授权的数据。
*   **定时任务**: 疫苗到期提醒定时任务默认每天上午 9 点运行，会检查未来 N 天内到期的疫苗并发送站内通知。

## 7. 常见问题

*   **无法连接数据库**: 请检查 `application.yml` 中的数据库连接信息是否正确，MySQL 服务是否已启动。
*   **前端页面空白或报错**: 请检查前端控制台是否有报错信息，确保后端服务已正常启动，且前端 `vite.config.ts` 中的代理配置正确。
*   **权限不足**: 请检查当前登录用户的角色是否具备操作权限。

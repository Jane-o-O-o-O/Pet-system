# 宠物信息管理系统（Pet Management System）开发规格说明（可直接给 Claude 开发）

> 目标：基于 **Spring Boot（后端） + Vue3 + Element Plus（前端） + MySQL（数据库）** 的前后端分离系统。  
> 角色：管理员 Admin、医护人员 Staff、宠物主人 Owner。  
> 重点：权限隔离（Owner 只能访问自己的数据）、医疗记录与疫苗提醒、寄养订单、统计报表。

---

## 0. 交付范围（MVP + 可扩展）

### MVP 必做
- 用户注册/登录（JWT）
- 角色权限（Admin/Staff/Owner）
- 宠物档案 CRUD（Owner 仅本人）
- 医疗记录 CRUD（Staff/Admin 可录入；Owner 可查看本人宠物）
- 疫苗记录（含 next_due_date + 定时提醒）
- 寄养订单管理（创建、状态流转、列表查询）
- Admin 数据统计（概览 + 简单聚合）
- 部署：后端 jar + 前端 dist + Nginx 反代

### 可选扩展（按时间）
- 站内消息中心（通知表）
- 短信/邮件提醒（第三方通道）
- 文件上传（医疗附件、宠物照片）
- Excel 导出报表

## 1. 技术栈与约束

### 后端
- Spring Boot 3.x
- Spring Web + Validation
- Spring Security + JWT
- MyBatis-Plus（推荐）或 JPA（二选一，默认 MyBatis-Plus）
- Spring Scheduler（定时提醒）
- MySQL 8.x

### 前端
- Vue 3 + Vite
- Element Plus
- Pinia
- Axios

### 统一约束
- RESTful API
- JSON 响应统一结构
- 分页统一：page/size + total
- 时间字段统一：ISO 8601（后端返回 `yyyy-MM-dd'T'HH:mm:ss`）

## 2. 角色与权限（RBAC + 数据级权限）

### 2.1 角色能力
- **Admin**
  - 用户/角色管理（创建 Staff、禁用账号）
  - 查看/管理所有宠物、医疗记录、疫苗、寄养订单
  - 统计报表
  - 系统配置（提醒默认提前天数等）
- **Staff**
  - 查看宠物（按业务：默认可查看全量或仅分配；MVP 简化为可查看全量）
  - 新增/更新医疗记录、疫苗记录
  - 更新寄养订单状态（确认/寄养中/完成/取消）
- **Owner**
  - 注册登录
  - 仅可 CRUD 自己的宠物
  - 仅可查看自己宠物的医疗/疫苗/寄养订单
  - 可创建寄养订单（与自己的宠物绑定）

### 2.2 数据级权限规则（必须实现）
- Owner 访问 `petId` 前，必须校验 `pet.owner_id == currentUser.id`
- Owner 查询列表时必须强制 `owner_id = currentUser.id`
- Owner 不允许访问他人医疗记录/订单（即使猜到 id）

## 4. 数据库设计（MySQL）

> 建议库名：`pet_mgmt`

### 4.1 表：sys_user
```sql
CREATE TABLE sys_user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  phone VARCHAR(20),
  email VARCHAR(100),
  role ENUM('ADMIN','STAFF','OWNER') NOT NULL,
  status TINYINT NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
````

### 4.2 表：pet

```sql
CREATE TABLE pet (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  owner_id BIGINT NOT NULL,
  name VARCHAR(50) NOT NULL,
  species VARCHAR(30) NOT NULL,
  breed VARCHAR(50),
  gender ENUM('M','F','U') DEFAULT 'U',
  birth_date DATE,
  weight DECIMAL(5,2),
  sterilized TINYINT DEFAULT 0,
  photo_url VARCHAR(255),
  remark TEXT,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_pet_owner_id(owner_id),
  INDEX idx_pet_species(species),
  CONSTRAINT fk_pet_owner FOREIGN KEY (owner_id) REFERENCES sys_user(id)
);
```

### 4.3 表：medical_record

```sql
CREATE TABLE medical_record (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  pet_id BIGINT NOT NULL,
  doctor_name VARCHAR(50),
  visit_date DATETIME NOT NULL,
  complaint TEXT,
  diagnosis TEXT,
  treatment TEXT,
  attachments JSON,
  created_by BIGINT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_mr_pet_date(pet_id, visit_date),
  CONSTRAINT fk_mr_pet FOREIGN KEY (pet_id) REFERENCES pet(id)
);
```

### 4.4 表：vaccine_record

```sql
CREATE TABLE vaccine_record (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  pet_id BIGINT NOT NULL,
  vaccine_name VARCHAR(100) NOT NULL,
  shot_date DATE NOT NULL,
  next_due_date DATE NOT NULL,
  remind_days_before INT DEFAULT NULL,
  remind_status ENUM('PENDING','SENT','DISABLED') NOT NULL DEFAULT 'PENDING',
  created_by BIGINT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_vr_due(next_due_date),
  CONSTRAINT fk_vr_pet FOREIGN KEY (pet_id) REFERENCES pet(id)
);
```

### 4.5 表：boarding_order

```sql
CREATE TABLE boarding_order (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(32) NOT NULL UNIQUE,
  pet_id BIGINT NOT NULL,
  owner_id BIGINT NOT NULL,
  staff_id BIGINT DEFAULT NULL,
  start_date DATE NOT NULL,
  end_date DATE NOT NULL,
  room_type VARCHAR(30),
  price_total DECIMAL(10,2) DEFAULT 0,
  status ENUM('CREATED','CONFIRMED','BOARDING','COMPLETED','CANCELLED') NOT NULL DEFAULT 'CREATED',
  remark TEXT,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_bo_owner(owner_id),
  INDEX idx_bo_pet(pet_id),
  INDEX idx_bo_status(status),
  CONSTRAINT fk_bo_pet FOREIGN KEY (pet_id) REFERENCES pet(id),
  CONSTRAINT fk_bo_owner FOREIGN KEY (owner_id) REFERENCES sys_user(id)
);
```

## 5. 后端 API 设计（REST）

### 5.1 通用响应

成功：

```json
{ "code": 0, "message": "OK", "data": null }
```

失败：

```json
{ "code": 40001, "message": "UNAUTHORIZED", "data": null }
```

### 5.2 认证（Auth）

* `POST /api/auth/register`（Owner 注册）

  * body: `{ "username": "...", "password": "...", "phone": "...", "email": "..." }`
* `POST /api/auth/login`

  * body: `{ "username": "...", "password": "..." }`
  * resp data: `{ "token": "...", "role": "OWNER", "userId": 1, "username":"..." }`

### 5.3 用户（Admin）

* `GET /api/users?page=&size=&keyword=`
* `POST /api/users`（Admin 创建 Staff/Owner）
* `PUT /api/users/{id}/status`（启用/禁用）
* `PUT /api/users/{id}/role`

### 5.4 当前用户

* `GET /api/users/me`

### 5.5 宠物（Pet）

* `POST /api/pets`
* `GET /api/pets?page=&size=&keyword=&species=`

  * Owner：强制仅返回本人
* `GET /api/pets/{id}`
* `PUT /api/pets/{id}`
* `DELETE /api/pets/{id}`

### 5.6 医疗记录（MedicalRecord）

* `POST /api/pets/{petId}/medical-records`
* `GET /api/pets/{petId}/medical-records?page=&size=`
* `GET /api/medical-records/{id}`
* `PUT /api/medical-records/{id}`
* `DELETE /api/medical-records/{id}`

权限：

* Staff/Admin：可新增/修改/删除（MVP）
* Owner：仅查看（本人宠物）

### 5.7 疫苗记录（VaccineRecord）

* `POST /api/pets/{petId}/vaccines`

  * body: `{ "vaccineName":"...", "shotDate":"2026-02-01", "nextDueDate":"2026-08-01", "remindDaysBefore": 7 }`
* `GET /api/pets/{petId}/vaccines?page=&size=`
* `PUT /api/vaccines/{id}`
* `DELETE /api/vaccines/{id}`
* `GET /api/reminds/vaccines?days=30`

  * 返回 Owner 自己宠物未来 N 天内到期列表（或 pending 列表）

### 5.8 寄养订单（BoardingOrder）

* `POST /api/boarding-orders`

  * body: `{ "petId":1, "startDate":"2026-03-01", "endDate":"2026-03-05", "roomType":"Standard", "priceTotal": 500, "remark":"" }`
* `GET /api/boarding-orders?page=&size=&status=`

  * Owner：仅本人
* `GET /api/boarding-orders/{id}`
* `PUT /api/boarding-orders/{id}/status`

  * body: `{ "status":"CONFIRMED" }`

状态机约束：

* CREATED → CONFIRMED → BOARDING → COMPLETED
* 任意状态可 → CANCELLED（但 COMPLETED 不可取消）
* 非法跳转返回 400

### 5.9 统计（Admin）

* `GET /api/admin/stats/overview`

  * data: `{ users, pets, medicalRecords, orders, vaccineDueIn30Days }`
* `GET /api/admin/stats/boarding?from=2026-01-01&to=2026-01-31`

  * data: `{ count, totalRevenue }`
* `GET /api/admin/stats/vaccine-due?days=30`

  * data: `{ dueCount, list:[...] }`

## 6. 定时任务：疫苗到期提醒（必须实现）

### 6.1 规则

* 每天固定时间扫描：`vaccine_record.next_due_date`
* 若 `next_due_date - remind_days_before <= today <= next_due_date`
* 且 `remind_status = PENDING`
* 则生成提醒：

  * MVP：写入 notification（站内消息）
  * 更新 `remind_status = SENT`
* 若 `remind_status = DISABLED` 则跳过

### 6.2 可配置项

* 默认提前天数：`sys_config.VACCINE_REMIND_DAYS_DEFAULT`（默认 7）
* cron：`sys_config.REMIND_JOB_CRON`（默认每天 09:00）

## 9. 业务校验（必须实现）

### 9.1 宠物

* name/species 必填
* Owner 新建宠物：owner_id = 当前用户
* Owner 更新/删除：必须是自己的 pet

### 9.2 医疗记录

* visit_date 必填
* pet_id 必须存在
* Owner 不允许创建/修改/删除（MVP 只给 Staff/Admin）

### 9.3 疫苗记录

* shot_date、next_due_date 必填
* `next_due_date >= shot_date`
* remind_days_before 若为空则取默认配置
* 修改 next_due_date 后重置 `remind_status = PENDING`（除非 DISABLED）

### 9.4 寄养订单

* `end_date >= start_date`
* 订单号 order_no：雪花/时间戳+随机（唯一）
* Owner 只能为自己的 pet 创建订单
* 状态机：非法跳转返回 400

## 10. 安全要求

* 密码：BCrypt（不允许明文/MD5）
* JWT：

  * 过期：建议 7 天
  * token payload：userId、role、username
* API 鉴权：

  * `/api/auth/**` 放行
  * 其余全部需要 token
* 日志：

  * 记录登录成功/失败、定时任务触发、关键写操作（可简化）

### 11.2 关键功能验收

* 注册→登录→进入角色首页
* Owner：创建宠物、查看详情、创建寄养订单

* 定时任务：到期前 N 天生成提醒并标记 SENT
